Venice API notes
Use the Venice docs:
OpenAI-compatible API
env var: VENICE_API_KEY
base URL: https://api.venice.ai/api/v1

Start with text-first support.
Do not block the first integration on full multimodal plumbing.

Product rules
Do not build:
a Venice tab
a Venice mode toggle
a whole-app Venice skin
a generic “private AI dashboard”
a giant provider/settings panel
Venice must appear as:
an agent capability
a private reasoning layer
a reasoning provenance / disclosure layer

Hashapp remains the product.

Goal
Ship the first Venice-backed UX slice that makes the product clearly eligible for the Venice track without derailing the core Hashapp story.
